## 1. Теория вероятностей

### 1.1 Три философии вероятности

Существуют три фундаментально разных взгляда на то, что такое вероятность. И это не абстрактный вопрос — от выбора философии зависит, как ты строишь и оцениваешь стратегии.

**Частотный подход (frequentist).** Вероятность — это предел частоты при бесконечном повторении эксперимента. "Вероятность орла = 0.5" означает, что при бесконечном числе бросков доля орлов стремится к 0.5. Этот подход стоит за классическими тестами гипотез (p-value, доверительные интервалы). Проблема для трейдинга: рынок — не повторяющийся эксперимент. Каждый день уникален. "Частота" в строгом смысле не определена.

**Байесовский подход.** Вероятность — это степень уверенности в гипотезе. "Вероятность 60%, что стратегия рабочая" — это мое текущее убеждение, которое я обновляю при получении новых данных. Байесовский подход более естественен для трейдинга: у тебя есть prior (начальные убеждения), приходят данные (бэктесты, live результаты), и ты обновляешь оценку. Цена — это формально коллективная апостериорная оценка стоимости актива.

**Субъективный подход (де Финетти).** Вероятность — это цена, которую ты готов заплатить за пари. Если ты готов поставить $60, чтобы выиграть $100, если событие произойдёт — ты оцениваешь вероятность в 60%. Этот подход напрямую связан с рынками: цена опциона — это, по сути, рыночная оценка вероятности.

На практике: используй частотный подход для тестирования гипотез (p-value, значимость), байесовский — для обновления убеждений о стратегиях и параметрах, субъективный — для интуиции о том, что означают рыночные цены.

### 1.2 Условная вероятность и ловушки мышления

Формула проста: P(A|B) = P(A∩B) / P(B). Но интуиция регулярно ошибается.

**Ошибка базовой частоты (base rate fallacy).** Допустим, у тебя есть индикатор, который правильно предсказывает обвал рынка в 90% случаев. Звучит отлично? Но обвалы случаются в ~2% дней. Считаем: из 1000 дней ~20 — обвалы, индикатор правильно ловит 18 из них. Но из 980 обычных дней он ложно срабатывает в 10% случаев = 98 ложных сигналов. Итого: 18 верных из 116 срабатываний = точность всего 15.5%. Индикатор с "90% accuracy" оказывается бесполезным, если базовая частота события низкая.

Это теорема Байеса в действии:
```
P(обвал | сигнал) = P(сигнал | обвал) × P(обвал) / P(сигнал)
                  = 0.90 × 0.02 / (0.90×0.02 + 0.10×0.98)
                  = 0.018 / 0.116 ≈ 15.5%
```

**Confusion of the inverse.** P(A|B) ≠ P(B|A). "80% обвалов произошли после инверсии кривой доходности" ≠ "после инверсии кривой доходности вероятность обвала 80%". Первое — P(инверсия | обвал) = 0.8. Второе — P(обвал | инверсия), которое зависит от того, как часто инверсии случаются без обвала.

**Simpson's paradox.** Стратегия может работать лучше конкурента в каждом отдельном секторе, но хуже в целом — если у неё другое распределение по секторам. Всегда разбивай результаты на подгруппы и проверяй, не исчезает ли эффект.

### 1.3 Математическое ожидание и его свойства

Математическое ожидание E[X] — это "средний результат в длинной серии". Для трейдера это ключевая метрика: если E[trade] > 0 (после всех комиссий), стратегия в долгосрочной перспективе прибыльна.

**Формула ожидаемой прибыли:**
```
E[trade] = win_rate × avg_win - (1 - win_rate) × avg_loss
```

Стратегия с win rate 30%, но средним выигрышем $500 и средним проигрышем $100 имеет E = 0.3×500 - 0.7×100 = $80 за сделку. Win rate переоценён. Важно отношение avg_win/avg_loss, а не доля побед.

**Линейность ожидания** — самое полезное свойство: E[X + Y] = E[X] + E[Y], *всегда*, даже для зависимых переменных. Ожидаемая доходность портфеля = взвешенная сумма ожидаемых доходностей, независимо от корреляций. Корреляции влияют на *дисперсию*, но не на *ожидание*.

**Jensen's inequality** — критически важна для понимания compounding. Если f — выпуклая функция, то E[f(X)] ≥ f(E[X]). Для вогнутых функций — наоборот. Логарифм — вогнутая функция, поэтому:

```
E[log(1 + r)] ≤ log(1 + E[r])
```

Это значит: **средняя геометрическая доходность всегда меньше арифметической.** Разница ≈ σ²/2 и называется "volatility drag". Два портфеля с одинаковой средней доходностью, но разной волатильностью будут расти по-разному — менее волатильный обгонит. При волатильности 20% в год volatility drag ≈ 2%. При 50% — уже 12.5%. При 100% — 50%, т.е. вы теряете половину средней доходности только на колебаниях. Это фундаментальная причина, почему leverage (увеличение позиции в долг) имеет естественный предел: в какой-то момент увеличение волатильности съедает весь дополнительный доход.

### 1.4 Закон больших чисел: медленнее, чем кажется

ЗБЧ утверждает: выборочное среднее сходится к истинному среднему при росте числа наблюдений. Но *как быстро*?

Стандартная ошибка среднего = σ / √n. Для оценки с точностью ε нужно n ≈ (σ/ε)² наблюдений.

**Конкретные числа для трейдинга:** типичная дневная волатильность акции ≈ 1.5%. Типичная дневная ожидаемая доходность ≈ 0.03% (8% годовых / 252). Отношение сигнал/шум = 0.03/1.5 = 0.02. Чтобы отличить эту доходность от нуля с 95% уверенностью, нужно n ≈ (1.96/0.02)² ≈ 9600 дней ≈ **38 лет**. Вот почему так трудно отличить хорошую стратегию от случайности.

Для стратегии с Sharpe 1.0 (signal/noise = 1/√252 ≈ 0.063 на дневной основе): нужно ≈ 960 дней ≈ 3.8 лет. Для Sharpe 2.0 — около года. Для Sharpe 0.5 — около 15 лет.

Эти числа объясняют многое:
- Почему менеджеров фондов нужно оценивать на горизонте 5-10 лет
- Почему большинство "работающих" стратегий на бэктесте — шум
- Почему высокочастотные стратегии (тысячи сделок в день) могут быть оценены за недели, а не годы

**Для толстых хвостов всё ещё хуже.** Если распределение имеет бесконечную дисперсию (об этом — в следующей главе), обычный ЗБЧ перестаёт работать. Сходимость идёт по другому закону (α-stable), и выборочное среднее может "прыгать" даже на огромных выборках.

### 1.5 Мартингалы и рыночная эффективность

**Мартингал** — процесс, для которого лучший прогноз будущего значения = текущее значение: E[X_{t+1} | вся информация до t] = X_t.

Гипотеза эффективного рынка (в слабой форме) утверждает, что цены (с поправкой на дрифт) — мартингал. Если это так, прошлые цены не содержат информации о будущих. Технический анализ бесполезен. Единственный способ заработать — брать на себя риск (risk premium) или иметь информационное преимущество.

**Optional Stopping Theorem** — одно из самых контринтуитивных следствий. Если процесс — мартингал, то *никакая* стоп-стратегия (стоп-лосс, тейк-профит, трейлинг стоп) не изменит ожидаемый результат. Это значит: если у стратегии E[trade] = 0, добавление стоп-лоссов не сделает её прибыльной. Стоп-лоссы меняют *распределение* результатов (чаще маленькие убытки, реже большие), но не *ожидание*.

Однако на практике рынки — не идеальные мартингалы. Слабые неэффективности существуют (иначе Renaissance Technologies не зарабатывал бы 66% годовых 30 лет подряд). Вся суть алготрейдинга — находить и эксплуатировать эти отклонения от мартингальности.

### 1.6 Теорема Байеса для трейдера

Байесовское обновление — это не просто формула, а образ мышления. Ты начинаешь с prior (начального убеждения), получаешь данные, и обновляешь убеждение.

**Пример: оценка стратегии.** До бэктеста ты считаешь, что стратегия рабочая с вероятностью 10% (prior — потому что большинство идей не работают). Бэктест показал Sharpe 2.0. Насколько сильно это должно обновить prior?

Зависит от *likelihood ratio*: насколько вероятнее увидеть Sharpe 2.0 для рабочей стратегии vs нерабочей? Если 50 стратегий уже тестировались, вероятность случайно получить Sharpe 2.0 хотя бы для одной из них — довольно высока. Prior update будет слабым. Если это была единственная тестируемая гипотеза с сильным экономическим обоснованием — update будет сильным.

Байесовский фреймворк формализует то, что хорошие трейдеры делают интуитивно: взвешивают новые данные с учётом контекста.

**Практический инструмент:** `PyMC` позволяет строить полноценные байесовские модели — вместо точечной оценки Sharpe получаешь *распределение* Sharpe с доверительными интервалами. Вместо "Sharpe = 1.5" получаешь "Sharpe = 1.5 ± 0.8 с 95% вероятностью", что намного информативнее.

## 2. Fat Tails: почему Гаусс врёт

Это, возможно, самая важная глава книги. Неправильное понимание хвостов распределения уничтожило больше капитала, чем любая другая математическая ошибка.

### 2.1 Что такое fat tails и почему они везде

Нормальное распределение — колоколообразная кривая — предсказывает, что экстремальные события практически невозможны. Правило 68-95-99.7: 99.7% значений в пределах ±3 стандартных отклонений. Событие в 5 стандартных отклонениях от среднего должно происходить раз в 14 000 лет. Событие в 6σ — раз в 1.5 миллиарда лет.

Реальные финансовые рынки генерируют 5σ-события несколько раз за десятилетие. 19 октября 1987 года S&P 500 упал на 20.5% за один день — это событие порядка 20σ. По нормальному распределению вероятность такого — примерно 10^(-87). Для сравнения, возраст Вселенной ≈ 10^10 лет. Нормальное распределение говорит: "Это не может произойти за всё время существования Вселенной." Рынок говорит: "Это произошло в обычный понедельник."

**Fat tails** (тяжёлые хвосты) означают, что экстремальные события происходят *намного* чаще, чем предсказывает нормальная модель. Это не аномалия и не "чёрный лебедь" — это фундаментальное свойство финансовых рынков (и многих других сложных систем).

### 2.2 Откуда берутся fat tails

Нормальное распределение возникает, когда сумма многих маленьких независимых случайных факторов (центральная предельная теорема). Финансовые рынки нарушают предпосылки ЦПТ несколькими способами:

**1. Кластеризация волатильности.** Периоды высокой волатильности сменяются периодами низкой. Это значит, что доходности *не* одинаково распределены: вчерашний большой ход увеличивает вероятность сегодняшнего большого хода (но не предсказывает направление). Формально — автокорреляция квадратов доходностей. Если смешать нормальные распределения с разной волатильностью, результат имеет толстые хвосты — даже если каждый режим по отдельности нормален.

**2. Положительная обратная связь.** Падение цен → margin calls → принудительные продажи → ещё падение. Или: рост цены → FOMO → покупки → ещё рост. Обратная связь усиливает движения и создаёт хвосты. В технических терминах: наблюдения не независимы, они влияют друг на друга.

**3. Информационные каскады.** Когда все смотрят на действия друг друга вместо собственного анализа, небольшое событие может спровоцировать лавину (herding). Математически — нарушение независимости.

**4. Дискретные события.** Earnings surprises, геополитические шоки, регуляторные решения — "прыжки" в цене, которые не описываются непрерывной диффузией.

**5. Нелинейность агрегации.** Даже если индивидуальные решения трейдеров распределены нормально, их агрегация через механизм ценообразования может порождать fat tails (через нелинейные эффекты ликвидности и market impact).

### 2.3 Количественная мера: kurtosis

**Kurtosis** (эксцесс) — четвёртый центральный момент, нормированный на дисперсию в квадрате:
```
κ = E[(X - μ)⁴] / σ⁴
```

Для нормального распределения κ = 3. Excess kurtosis = κ - 3 — сколько "лишних" хвостов сверх нормального. Типичные дневные доходности акций имеют excess kurtosis 5-20 (на порядок больше нуля). Для 5-минутных данных — ещё выше.

**Проблема с kurtosis:** это единственное число, и оно может обмануть. Два распределения с одинаковым kurtosis могут иметь совершенно разную форму хвостов (один симметричный, другой с длинным левым хвостом). Для полного понимания нужно смотреть на:
- **Skewness** (асимметрия, 3-й момент) — насколько хвосты несимметричны. Отрицательный skew типичен для акций (редкие крупные обвалы). Положительный — для стратегий покупки опционов
- **QQ-plot** (quantile-quantile plot) — визуальное сравнение с нормальным распределением. Если точки отклоняются от прямой в хвостах — fat tails. Инструмент: `scipy.stats.probplot(returns, plot=plt)`. Один график стоит больше, чем десять числовых метрик
- **Tail index** — более фундаментальная характеристика, см. далее

### 2.4 Распределения с тяжёлыми хвостами

Иерархия распределений от "тонких" к "толстым" хвостам:

**Нормальное (Gaussian):** хвосты убывают как exp(-x²/2). Супер-быстро. Все моменты конечны. Это *единственное* распределение, которое полностью характеризуется средним и дисперсией. Если кто-то говорит "мы знаем среднее и дисперсию, этого достаточно" — он неявно предполагает нормальность.

**Распределение Стьюдента (t-distribution):** хвосты убывают как степенная функция x^(-ν-1), где ν — степени свободы. При ν > 30 практически неотличимо от нормального. При ν = 4-5 — хорошее приближение для дневных доходностей акций. При ν ≤ 4 — бесконечный kurtosis (четвёртый момент не существует). При ν ≤ 2 — бесконечная дисперсия (второй момент не существует, и стандартное отклонение бессмысленно). При ν ≤ 1 — Коши, даже среднее не существует. Для финансов типичная оценка ν ≈ 3-6. Это означает, что kurtosis формально бесконечен или очень нестабилен — любая выборочная оценка ненадёжна.

**α-stable (Lévy-stable) распределения:** обобщение нормального, с параметром α ∈ (0, 2]. При α = 2 → нормальное. При α < 2 → бесконечная дисперсия. При α ≤ 1 → бесконечное среднее. Мандельброт в 1960-х предложил α ≈ 1.7 для хлопковых фьючерсов. Для акций — оценки варьируются от 1.5 до 1.9. Ключевое свойство: α-stable распределения — единственные, которые сохраняют форму при сложении (как нормальное, но с толстыми хвостами). Проблема: при α < 2 дисперсия бесконечна, что делает Sharpe ratio, VaR и большинство стандартных метрик *формально бессмысленными*.

**Generalized Pareto Distribution (GPD):** не для всего распределения, а только для хвоста. Теорема Пикандса-Балкемы-де Хаана (аналог ЦПТ для экстремумов) утверждает: при достаточно высоком пороге хвосты *любого* распределения можно приблизить GPD. Параметр формы ξ: при ξ > 0 — тяжёлые хвосты (Pareto-type), при ξ = 0 — экспоненциальные (как у нормального), при ξ < 0 — ограниченные. Для финансов ξ > 0 практически всегда.

### 2.5 Extreme Value Theory (EVT)

EVT — это теория экстремальных значений, целая область математики, специально созданная для моделирования редких событий. Она отвечает на вопрос: "Какова вероятность события, которого мы ещё не наблюдали?"

**Две основных теоремы:**

**Fisher-Tippett-Gnedenko (для блочных максимумов):** максимум из n iid наблюдений (при нормировке) сходится к одному из трёх распределений: Gumbel, Fréchet или Weibull. Для финансов — обычно Fréchet (тяжёлые хвосты).

**Pickands-Balkema-de Haan (для превышений порога):** распределение значений выше высокого порога → GPD. Это подход Peaks Over Threshold (POT), более популярный на практике — не нужно делить на блоки, используешь все экстремальные наблюдения.

**Зачем EVT трейдеру:**
- Оценка VaR на уровнях 99.9%, 99.99% — там, где исторических данных недостаточно
- Стресс-тестирование: "Что если случится событие хуже всего, что было?"
- Оценка вероятности "невозможных" сценариев (которые оказываются не такими невозможными)

**Инструменты:** `scipy.stats.genpareto` для подбора GPD, пакет `pyextremes` для полного анализа EVT.

### 2.6 Практические последствия fat tails

**1. VaR на основе нормального распределения занижает риски.** Если банк считает VaR(99%) = $10M, используя нормальное распределение, реальный VaR может быть $30-50M. Это не теоретическая проблема — это причина, по которой банки регулярно теряют "невозможные" суммы.

**2. Среднее и дисперсия могут быть нестабильны.** Если tail index α < 4 (что обычно для финансов), выборочный kurtosis нестабилен. Если α < 2 (что бывает для отдельных акций и крипто), даже выборочная дисперсия нестабильна — она скачет от выборки к выборке. Это означает, что волатильность и Sharpe ratio, посчитанные по разным периодам, могут сильно отличаться *не потому что рынок изменился*, а потому что оценки ненадёжны.

**3. Диверсификация работает хуже, чем кажется.** По нормальной модели, диверсификация по 30 акциям убирает ~95% идиосинкратического риска. При fat tails этот эффект слабее — экстремальные события в одной акции "пробивают" через портфель. Нужно больше позиций для того же уровня диверсификации.

**4. Leverage смертельно опасен.** При нормальном распределении 3x leverage на S&P500 выглядит агрессивно, но терпимо. При fat tails однодневный ход в 7% (который случается раз в несколько лет) при 3x leverage = -21% за день. Ход в 15% (1987, 2020) при 3x = -45% за день, и это после того, как маржин-колл принудительно закроет позицию по худшим ценам.

**5. Backtesting оптимистичен.** Бэктест на 20 годах данных может не содержать ни одного 6σ-события — просто потому что 20 лет мало. Стратегия, которая "идеально работала" 20 лет, может потерять всё в одном хвостовом событии. Тестировать нужно не только на исторических кризисах, но и на синтетических сценариях, сгенерированных с реалистичными fat tails.

### 2.7 Что с этим делать

**Используй t-распределение вместо нормального** для моделирования доходностей. Подбери степени свободы ν из данных (`scipy.stats.t.fit(returns)`). Это минимальное изменение, которое существенно улучшает модели.

**Моделируй хвосты отдельно (EVT/GPD).** Для оценки экстремальных рисков не полагайся на параметрическое распределение "тела" — оно не знает про хвосты. Используй POT-метод для хвостов.

**Используй робастные оценки.** Медиана вместо среднего, MAD (median absolute deviation) вместо стандартного отклонения, Winsorized mean. Они менее чувствительны к выбросам.

**Думай о стресс-сценариях.** "Что произойдёт с портфелем, если завтра случится событие 10σ?" — не параноидальный вопрос, а разумное планирование. Убедись, что ответ — "я потеряю X% и продолжу работать", а не "я потеряю всё".

**Разделяй "нормальные" и "кризисные" модели.** Один набор параметров для спокойного рынка, другой — для кризиса. Hidden Markov Models, GARCH с переключением режимов и т.д.

## 3. Статистика и проверка гипотез

### 3.1 p-value: что это на самом деле

p-value — вероятность получить результат *не менее экстремальный*, чем наблюдаемый, *при условии что нулевая гипотеза верна*.

Что p-value **не означает:**
- Вероятность, что гипотеза верна (это байесовская posterior, совершенно другая вещь)
- Вероятность, что результат — случайность (это P(H0), а p-value = P(data|H0))
- Размер эффекта (p может быть 0.0001 для бесполезно малого эффекта)

**Пример:** стратегия на 5 годах дневных данных показала среднюю доходность 0.02%/день при стандартном отклонении 1%. t-статистика = 0.02 / (1 / √1260) ≈ 0.71. p-value ≈ 0.48. Мы не можем отвергнуть H0 (доходность = 0). Но это не значит, что стратегия не работает — у нас просто недостаточно данных. 0.02%/день = 5% годовых — вполне полезная стратегия, которую мы не можем *статистически подтвердить* с 5 годами данных.

### 3.2 Множественное тестирование: проблема #1 в алготрейдинге

Если ты тестируешь одну стратегию и получаешь p < 0.05 — вероятность ложного срабатывания 5%. Если тестируешь 100 стратегий — ожидай 5 ложно-положительных. Если тестируешь 1000 — 50.

**Data mining bias** — системная ошибка, возникающая из-за поиска по множеству гипотез без коррекции. Это самая частая причина "стратегий, которые работали на бэктесте, но сломались в live". Ты не "обнаружил паттерн" — ты *перебрал достаточно паттернов*, чтобы найти случайно работающий.

**Методы коррекции:**

**Bonferroni** — самый простой и консервативный: делит α на N тестов. Если α = 0.05 и N = 100, порог = 0.0005. Проблема: слишком строг, пропускает реальные эффекты.

**Benjamini-Hochberg (FDR)** — контролирует долю ложных среди отвергнутых (False Discovery Rate), а не вероятность хотя бы одного ложного. Менее строг, лучше для exploratory analysis.

**Deflated Sharpe Ratio (DSR)** — метод Бейли и Лопеса де Прадо, специально разработанный для оценки торговых стратегий. Суть: если ты протестировал N стратегий и лучшая показала Sharpe = S, какова вероятность, что это не лучший из N случайных результатов? Учитывает skewness и kurtosis доходностей.

Формула: DSR сравнивает наблюдаемый Sharpe с ожидаемым максимумом из N испытаний. Если N = 500 и доходности нормальны, ожидаемый максимальный Sharpe чисто случайно ≈ 3.3 (для 5 лет дневных данных). То есть Sharpe 3.0 после 500 тестов — скорее всего шум!

**Практическое правило:** отслеживай *каждую* протестированную гипотезу (даже неудачные). Честная статистика стратегии включает N_trials. Инструмент: `statsmodels.stats.multitest.multipletests`.

### 3.3 Effect size и практическая значимость

p-value говорит, реален ли эффект. Effect size говорит, *насколько* он велик. Для трейдинга второе важнее.

Стратегия с p = 0.001 и средней доходностью $0.50 за сделку при стоимости исполнения $5 за сделку — статистически блестящая и практически бесполезная.

**Cohen's d** = разница средних / общее стандартное отклонение. В контексте трейдинга — это по сути Sharpe ratio на другом масштабе.

**Power analysis** — обратная задача: сколько данных нужно, чтобы обнаружить эффект заданного размера? Для Sharpe = 0.5 при α = 0.05 и power = 0.8 нужно ≈ 1570 дневных наблюдений (≈ 6 лет). Для Sharpe = 1.0 — ≈ 400 наблюдений (≈ 1.5 года). Для Sharpe = 2.0 — ≈ 100 (≈ 5 месяцев). Инструмент: `statsmodels.stats.power.TTestPower`.

Если у тебя 2 года данных и стратегия показала Sharpe 0.3, не тратить время на статистические тесты — у тебя заведомо нет мощности для обнаружения такого маленького эффекта. Нужно или больше данных, или более частая торговля (intraday → больше наблюдений).

### 3.4 Непараметрические методы

Параметрические тесты (t-тест, F-тест) предполагают конкретное распределение (обычно нормальное). При fat tails — предположение нарушено, тесты могут давать ложные результаты.

**Bootstrap** — универсальный швейцарский нож. Идея: у тебя есть одна выборка. Ты повторно семплируешь из неё *с возвращением*, создавая тысячи "виртуальных выборок". Для каждой считаешь нужную статистику (Sharpe, max drawdown, что угодно). Получаешь распределение статистики, из которого строишь доверительные интервалы.

Преимущества: не нужно знать распределение, работает для *любой* статистики. Недостаток: предполагает, что выборка репрезентативна (если в данных не было кризисов, bootstrap не "придумает" их).

**Block bootstrap** — для временных рядов. Обычный bootstrap разрушает автокорреляцию (перемешивает порядок). Block bootstrap семплирует *блоками* (например, по 20 дней), сохраняя внутреннюю структуру. Критично для любых метрик, чувствительных к порядку (max drawdown, run length и т.д.).

**Permutation tests** — для сравнения двух групп. Идея: перемешиваем метки (какое наблюдение к какой группе), вычисляем разницу. Повторяем 10000 раз. Если реальная разница в крайних процентах распределения перемешанных — она значима. Полностью непараметрический метод.

### 3.5 Байесовская vs частотная статистика

| Аспект | Частотная | Байесовская |
|--------|-----------|------------|
| Что оценивает | P(data \| H0) | P(H \| data) |
| Результат | p-value, CI | Posterior distribution |
| Prior knowledge | Не используется | Встроена (prior) |
| Множественное тестирование | Нужна коррекция | Автоматически через prior |
| Малые выборки | Плохо работает | Работает (prior помогает) |
| Интерпретация | Контринтуитивная | Естественная |
| Вычислительная стоимость | Быстро | Дорого (MCMC) |

Для трейдинга байесовский подход часто лучше: он естественно включает prior (большинство стратегий не работают), даёт распределение параметров (а не точечные оценки), и не требует мучительной коррекции на множественное тестирование.

**Инструменты:** `PyMC` (MCMC, полноценные модели), `arviz` (визуализация), `bambi` (байесовская регрессия с formula interface).

## 4. Sharpe Ratio и метрики эффективности

### 4.1 Sharpe Ratio: определение и интуиция

Sharpe ratio = (доходность - безрисковая ставка) / волатильность. Это *доходность на единицу риска*. Если Sharpe = 1, ты зарабатываешь 1% дополнительной доходности на каждый 1% волатильности.

**Годовой Sharpe из дневных данных:**
```
SR_annual = SR_daily × √252 = (mean_daily / std_daily) × √252
```

Множитель √252 — следствие того, что при iid доходностях среднее масштабируется как T, а std — как √T, поэтому отношение масштабируется как √T.

### 4.2 Ориентиры

| Sharpe | Качество | Примеры |
|--------|----------|---------|
| < 0 | Убыточная | — |
| 0 - 0.5 | Слабая | Большинство "стратегий" после комиссий |
| 0.5 - 1.0 | Приемлемая | Средний хедж-фонд (до комиссий ≈ 1.5) |
| 1.0 - 2.0 | Хорошая | Хорошие систематические стратегии |
| 2.0 - 3.0 | Отличная | Топ-квантовые фонды |
| > 3.0 | Подозрительная | Либо гений, либо ошибка, либо data mining |

**Важный контекст:** Sharpe > 3.0 на дневных данных за 3+ лет — красный флаг. Нужно тщательно проверить на look-ahead bias, survivorship bias и множественное тестирование. Renaissance Medallion Fund имеет Sharpe ≈ 6 (до комиссий, ≈ 2.5 после) — но это *исключение*, подтверждающее правило, с уникальными данными, инфраструктурой и >100 PhD.

### 4.3 Проблемы Sharpe Ratio

**1. Предполагает симметрию.** Sharpe наказывает одинаково за волатильность вверх и вниз. Но стратегия с волатильностью только вверх (редкие большие выигрыши, частые маленькие проигрыши) — хорошая, хотя Sharpe может быть средним. Стратегия с волатильностью только вниз (частые маленькие выигрыши, редкие обвалы) — опасная, хотя Sharpe может быть высоким. Продажа put-опционов имеет красивый Sharpe до тех пор, пока рынок не обваливается.

**2. Предполагает нормальность.** Sharpe полностью характеризует распределение только если оно нормальное. При fat tails или skewness информации недостаточно. Два портфеля с одинаковым Sharpe могут иметь радикально разные risk profiles.

**3. Манипулируем.** Sharpe можно искусственно завысить:
- Сглаживание доходностей (illiquid assets, mark-to-model вместо mark-to-market)
- Продажа хвостовых рисков (высокий Sharpe до взрыва)
- Написание out-of-the-money опционов (собираешь премии, Sharpe растёт, пока не случится кризис)
- Leverage с rebalancing на дневной основе (compounding скрывает реальную волатильность)

**4. Нестабилен при fat tails.** Если tail index α < 4 (что типично), четвёртый момент бесконечен, и стандартная ошибка оценки Sharpe — тоже бесконечна. На практике это проявляется как большая вариабельность Sharpe от периода к периоду, даже если стратегия не меняется. Доверительный интервал для Sharpe = 1.0 на 5 годах дневных данных — примерно [0.2, 1.8] даже для нормальных доходностей. При fat tails — ещё шире.

### 4.4 Альтернативные метрики

**Sortino Ratio** = (доходность - target) / downside deviation. Downside deviation считает только отрицательные отклонения. Не наказывает за "хорошую" волатильность. Более осмысленная метрика для стратегий с позитивным skew.

**Calmar Ratio** = годовая доходность / max drawdown. Отражает то, что трейдера реально волнует: "Сколько я заработаю относительно максимальной просадки?" Проблема: max drawdown — единственное число (наихудший случай), что делает метрику нестабильной — один плохой день сильно меняет Calmar.

**Omega Ratio** = площадь gain distribution выше порога / площадь loss distribution ниже порога. Использует *всё* распределение, а не только первые два момента. Omega > 1 — стратегия лучше порога. Не предполагает нормальность. Недостаток: менее интуитивна.

**CAGR/Max Drawdown** = Compound Annual Growth Rate / Max Drawdown. Просто и информативно. Если CAGR = 15% и max DD = 30%, то CAGR/MaxDD = 0.5. Это говорит: "Ты заработаешь 15% в год, но в процессе потеряешь 30% от пика." Если это неприемлемо — уменьши позицию.

**Information Ratio** = (доходность портфеля - доходность бенчмарка) / tracking error. Sharpe относительно бенчмарка. Для long-only стратегий часто более релевантен, чем абсолютный Sharpe.

**Tail Ratio** = 95th percentile / |5th percentile|. Показывает симметрию хвостов. > 1 — правый хвост длиннее (хорошо). < 1 — левый длиннее (опасно).

### 4.5 Статистическая значимость Sharpe

Стандартная ошибка Sharpe (при нормальных доходностях):
```
SE(SR) ≈ √((1 + SR²/2) / n)
```

Для SR = 1.0 и n = 252 (1 год): SE ≈ √(1.5/252) ≈ 0.077, то есть 95% CI ≈ [0.85, 1.15]. Для n = 1260 (5 лет): SE ≈ 0.035, CI ≈ [0.93, 1.07]. Разумная точность требует лет данных.

При ненормальных доходностях (skew s, excess kurtosis k):
```
SE(SR) ≈ √((1 + 0.5·SR² - s·SR + (k/4)·SR²) / n)
```

При отрицательном skew и высоком kurtosis (типично для акций) SE растёт — Sharpe оценивается менее точно.

**Минимальный track record для статистической значимости (Bailey & Lopez de Prado):**
```
n_min = (z_α / SR)² × (1 + (k-1)/4 × SR²)
```

Для Sharpe = 0.5 и нормальных доходностей: n_min ≈ 62 месяцев ≈ 5 лет. Для Sharpe = 1.0: ≈ 16 месяцев. Для Sharpe = 2.0: ≈ 4 месяца.

### 4.6 Profit Factor и другие торговые метрики

**Profit Factor** = Σ(прибыльных сделок) / |Σ(убыточных сделок)|. Если > 1, стратегия прибыльна. 1.5 — нормально, 2.0 — хорошо, 3.0+ — отлично или подозрительно.

**Payoff Ratio** = средний выигрыш / средний проигрыш. Показывает, "стоят ли свеч" сделки.

**Recovery Factor** = чистая прибыль / max drawdown. Сколько раз стратегия "отбила" свою худшую просадку.

Эти метрики дополняют друг друга. Опасно ориентироваться на одну — нужен dashboard из 4-5 ключевых:
- Sharpe (или Sortino) — risk-adjusted return
- Max drawdown — worst case
- Profit factor — general profitability
- Win rate + payoff ratio — характер сделок
- Recovery factor — resilience

## 5. Корреляции: что они скрывают

### 5.1 Что корреляция на самом деле измеряет

Корреляция Пирсона r ∈ [-1, 1] — мера *линейной* зависимости. Если r = 0, переменные *линейно* независимы, но могут быть сильно *нелинейно* зависимы. Классический пример: X ~ Normal(0,1), Y = X². Корреляция r = 0, но Y полностью определяется X.

Для финансов это не академическая проблема. Зависимость волатильности от доходности — нелинейная (volatility smile). Зависимость между активами в хвостах — нелинейная (tail dependence). Корреляция слепа к этим зависимостям.

### 5.2 Корреляция не постоянна

Одно из самых опасных предположений — что корреляция стабильна. В реальности:

**Корреляция растёт в кризис.** Это не просто наблюдение, а математический факт: если два актива имеют совместное распределение с "хвостовой зависимостью" (tail dependence > 0), то их условная корреляция в нижнем хвосте выше, чем безусловная. Практически все пары акций демонстрируют это свойство.

В терминах портфельной теории это значит: диверсификация работает хуже всего именно тогда, когда она нужна больше всего. Портфель из 30 "слабо коррелированных" акций в спокойное время (r ≈ 0.3) может показывать корреляцию r ≈ 0.8 во время обвала. Эффект диверсификации уменьшается в 3-4 раза.

**Корреляция зависит от горизонта.** Два актива могут быть некоррелированы на дневном горизонте и сильно коррелированы на месячном (или наоборот). Типичный пример: акции и облигации слабо коррелированы на коротких горизонтах, но сильно (отрицательно) — на длинных.

**Корреляция зависит от режима.** В bull market корреляции между акциями обычно ниже (каждая растёт "по своим причинам"). В bear market — выше ("всё продают"). Скользящая корреляция — минимальный инструмент мониторинга, но она лагает (использует устаревшие данные). DCC-GARCH — динамическая модель, которая адаптируется быстрее.

### 5.3 Корреляция ≠ причинность (и что вместо неё)

Корреляция не отвечает на вопрос "X вызывает Y?" Она говорит: "X и Y двигаются вместе." Причины совместного движения:
- X → Y (X вызывает Y)
- Y → X (Y вызывает X)
- Z → X и Z → Y (общий скрытый фактор)
- Совпадение (spurious correlation — число утопленников коррелирует с продажами мороженого)

**Granger causality** — тест на *предсказательную* причинность: "помогают ли прошлые значения X предсказать Y сверх прошлых значений самого Y?" Это не "настоящая" причинность, а направленная предсказуемость. Но для трейдинга этого достаточно — нам важно предсказание, а не метафизика. Инструмент: `statsmodels.tsa.stattools.grangercausalitytests`.

**Transfer entropy** — нелинейная версия Granger causality (подробнее в главе про энтропию).

### 5.4 Альтернативы корреляции Пирсона

**Корреляция Спирмена** — ранговая корреляция. Вместо значений использует ранги. Устойчива к выбросам, ловит монотонные (но не обязательно линейные) зависимости. Для финансовых данных часто лучше Пирсона.

**Корреляция Кендалла (τ)** — основана на конкордантных/дискордантных парах. Ещё более устойчива. Имеет красивую связь с копулами: τ = 2·arcsin(r)/π для Гауссовой копулы. Отличие τ от этого значения — признак нелинейной зависимости.

**Distance correlation (dCor)** — единственная мера, для которой dCor = 0 ⟺ полная независимость (не только линейная). Если Pearson r = 0 но dCor > 0 — есть нелинейная зависимость. Вычислительно дороже, но информативнее. Инструмент: `dcor.distance_correlation`.

**Mutual Information (MI)** — мера из информационной теории, ловит *любой* тип зависимости (подробнее в главе 8). MI = 0 ⟺ полная независимость. Проблема: оценка MI из конечных данных ненадёжна.

### 5.5 Копулы: зависимость за пределами корреляции

Корреляция описывает только одну грань зависимости (линейную, в "теле" распределения). Полная зависимость описывается *копулой* — функцией, которая связывает маргинальные распределения в совместное.

**Теорема Склара:** любое совместное распределение можно разложить как маргиналы + копула. Маргиналы описывают каждую переменную по отдельности. Копула описывает структуру зависимости.

Практический смысл: ты можешь иметь любые маргиналы (t-распределение, GPD — что хочешь) и *отдельно* моделировать зависимость (какой копулой связать).

**Типы копул:**

**Гауссова копула** — зависимость как у многомерного нормального. *Не имеет tail dependence*: экстремальные события в разных активах "не знают" друг о друге. Это копула, которую Дэвид Ли использовал для оценки CDO, и которая привела к кризису 2008 — она систематически занижала вероятность совместных дефолтов.

**t-копула** — как Гауссова, но с параметром ν (степени свободы). При малых ν имеет *значительную tail dependence*: экстремальные события коррелированы. Значительно лучше описывает финансовые рынки.

**Clayton** — нижняя tail dependence (совместные падения). Подходит для моделирования кризисных сценариев.

**Gumbel** — верхняя tail dependence (совместные ралли).

**Коэффициент tail dependence** λ — ключевая метрика: "Если актив A упал ниже 1-го перцентиля, какова вероятность, что B тоже?" Для Гауссовой копулы λ = 0 (всегда). Для t-копулы λ > 0 (зависит от ν и корреляции). Для финансовых данных эмпирическая λ обычно 0.2-0.5, что означает: при экстремальном падении одного актива другой тоже падает экстремально с вероятностью 20-50%. Это *намного* выше, чем предсказывает обычная корреляция.

**Инструменты:** `copulas` (Python), `VineCopula` (R), `statsmodels` (некоторые).

### 5.6 Ковариационная матрица: проблема размерности

Для N активов ковариационная матрица имеет N(N+1)/2 уникальных элементов. При N = 500 акциях — это 125 250 параметров, оцениваемых из, скажем, 252 × 5 = 1260 дневных наблюдений. Оценки крайне ненадёжны.

**Random Matrix Theory (RMT)** даёт элегантное решение. Теорема Марченко-Пастура: для матрицы случайных (шумовых) данных собственные значения распределены между λ_- = (1 - √(N/T))² и λ_+ = (1 + √(N/T))². Собственные значения *реальной* корреляционной матрицы, попадающие в этот интервал — скорее всего шум. Только те, что выше λ_+ — содержат реальную информацию (обычно это рыночный фактор и несколько секторных).

**Shrinkage (сжатие Ledoit-Wolf):** компромисс между выборочной ковариационной матрицей (ненадёжна, но специфична) и целевой матрицей (надёжна, но неспецифична — например, единичная). Оптимальный коэффициент сжатия α ∈ [0, 1] минимизирует ожидаемую ошибку. Инструмент: `sklearn.covariance.LedoitWolf`.

## 6. Временные ряды

### 6.1 Стационарность: фундаментальное требование

Подавляющее большинство статистических методов предполагают, что данные — из одного и того же распределения. Для временных рядов это формализуется как *стационарность*: среднее, дисперсия и автоковариационная функция не меняются со временем.

Цены акций — *не* стационарны. Они растут (тренд), их волатильность меняется. Доходности (r_t = log(P_t/P_{t-1})) — *приблизительно* стационарны, и это основная причина, почему все работают с доходностями, а не ценами.

**Почему log-доходности, а не простые:**
- Аддитивность: log-return за 2 дня = сумма дневных log-returns
- Непрерывная доходность: связана с GBM через стохастическое исчисление
- Симметрия: +5% и -5% одинаковы по модулю (для простых — нет: потеря 50% требует роста на 100%)
- При малых r: log(1+r) ≈ r, разница несущественна

**Тесты стационарности:**
- **ADF (Augmented Dickey-Fuller):** H0 = ряд нестационарен. Если p < 0.05 → стационарен
- **KPSS:** H0 = ряд стационарен (обратная H0!). Если p < 0.05 → нестационарен
- **Использовать оба:** если ADF говорит "стационарен" и KPSS говорит "стационарен" → уверены. Если расходятся — возможно, тренд-стационарен или есть структурный слом

**Инструменты:** `statsmodels.tsa.stattools` (adfuller, kpss), `arch.unitroot` (Phillips-Perron, DFGLS, Zivot-Andrews).

### 6.2 Stylized facts финансовых временных рядов

Это эмпирические закономерности, наблюдаемые *на всех рынках, во все эпохи, для всех активов*. Любая модель, которая их не воспроизводит — неадекватна.

**1. Отсутствие автокорреляции доходностей.** ACF(r_t, r_{t-k}) ≈ 0 для всех k > 0. Доходности непредсказуемы по прошлым доходностям (иначе было бы слишком просто).

**2. Автокорреляция абсолютных/квадратичных доходностей (volatility clustering).** ACF(|r_t|, |r_{t-k}|) > 0 и убывает медленно (может быть значимой для k > 100 дней). Большие движения порождают большие движения (но не в определённом направлении). Это *главный* stylized fact — он порождает fat tails, skewness и многие другие эффекты.

**3. Fat tails.** (Подробно в главе 2.)

**4. Leverage effect.** Отрицательные шоки увеличивают волатильность сильнее, чем положительные шоки того же размера. corr(r_t, |r_{t+k}|) < 0 для k > 0. Объяснение: падение цены → увеличение leverage → увеличение риска → увеличение волатильности.

**5. Asymmetric tail.** Левый хвост (убытки) обычно толще правого (прибыли). Negative skewness типична для акций.

**6. Aggregational Gaussianity.** По мере увеличения горизонта (от тиков к минутам, дням, месяцам) распределение стремится к нормальному. Это ЦПТ в действии, хотя сходимость медленная.

**7. Volume-volatility correlation.** Высокая волатильность сопровождается высоким объёмом торгов.

### 6.3 Модели волатильности

Волатильность — *единственная* характеристика финансовых рынков, которая предсказуема с разумной точностью. Доходности — почти нет. Волатильность — да. Это делает модели волатильности одним из самых полезных инструментов.

**GARCH(1,1):** σ²_t = ω + α·ε²_{t-1} + β·σ²_{t-1}. Волатильность зависит от вчерашнего шока (α) и вчерашней волатильности (β). α + β < 1 для стационарности. На практике α ≈ 0.05-0.15, β ≈ 0.80-0.95. Persistence α + β ≈ 0.95-0.99 (волатильность очень инертна).

**EGARCH** — экспоненциальный GARCH. Логарифмирует волатильность, что (1) гарантирует σ² > 0 без ограничений, (2) позволяет моделировать *leverage effect* (разный отклик на положительные и отрицательные шоки).

**GJR-GARCH** — добавляет индикаторную функцию для отрицательных шоков: σ²_t = ω + (α + γ·I_{ε<0})·ε²_{t-1} + β·σ²_{t-1}. Если γ > 0, отрицательные шоки увеличивают волатильность сильнее.

**HAR-RV** — Heterogeneous Autoregressive Realized Volatility. Не GARCH-семейство, а простая регрессия: RV_t = c + β_d·RV_{t-1} + β_w·RV_{week} + β_m·RV_{month}. Три компоненты — дневная, недельная, месячная — отражают гипотезу "гетерогенных рыночных участников" (day traders, swing traders, institutions). Простая модель, но часто конкурирует с GARCH. Работает с *realized volatility* (из внутридневных данных), что точнее, чем daily close-to-close.

**Инструмент:** `arch` (Python) — GARCH, EGARCH, GJR-GARCH и многие другие.

### 6.4 Коинтеграция: основа pairs trading

Два ряда *коинтегрированы*, если их линейная комбинация стационарна. Каждый по отдельности может быть нестационарным (расти, падать), но их разница "ходит вокруг среднего" — mean-reverts.

Классическая аналогия — пьяный с собакой на поводке. Оба идут случайно (нестационарны), но расстояние между ними ограничено (стационарно). Аналогия на рынке: Coca-Cola и PepsiCo — каждая движется по-своему, но разница стабильна в долгосрочной перспективе.

**Разница между корреляцией и коинтеграцией:**
- Высокая корреляция = двигаются в одном направлении *одновременно*
- Коинтеграция = линейная комбинация стационарна *в долгосрочной перспективе*

Два ряда могут быть коинтегрированы, но слабо коррелированы (на коротких горизонтах движутся по-разному, но не расходятся далеко). И наоборот: сильно коррелированы, но не коинтегрированы (движутся вместе, но могут разойтись навсегда).

**Half-life** — время, за которое спред уменьшится вдвое. Определяет практическую торговабельность: если half-life = 3 дня — стратегия быстрая. Если 60 дней — капитал заморожен надолго. Если > 100 дней — вероятно, не стоит.

**Инструменты:** `statsmodels.tsa.stattools.coint` (Engle-Granger), `statsmodels.tsa.vector_ar.vecm.coint_johansen` (Johansen).

### 6.5 Fractional differentiation

Проблема: цены нестационарны (d = 0), доходности стационарны (d = 1). Но при d = 1 мы *теряем информацию* о ценовых уровнях. Акция, стоящая $10 и $1000, будет иметь одинаковые доходности — но уровень цены может нести информацию (например, о поддержке/сопротивлении).

Идея Lopez de Prado: используем дробное d ∈ (0, 1). Минимальное d, при котором ряд становится стационарным. Это сохраняет максимум "памяти" (корреляция с исходным ценовым рядом 80-95%) при стационарности.

На практике: оптимальное d обычно 0.3-0.5 для акций. Фракционально дифференцированный ряд — лучший входной признак для ML-моделей, чем цены (нестационарны) или доходности (слишком много информации потеряно).

**Инструмент:** можно реализовать через свёртку с весами w_k, или использовать `fracdiff` пакет.

### 6.6 Спектральный анализ

Альтернативный взгляд на ряд: вместо "значение по времени" — "мощность по частоте". Периодограмма / спектральная плотность показывает, какие частоты (циклы) доминируют.

Для финансовых данных обычно нет ярко выраженных пиков (рынок — не синусоида). Но иногда обнаруживаются: сезонные паттерны (годовой цикл, end-of-month, day-of-week), циклы волатильности.

**Wavelet-анализ** — обобщение, которое даёт разложение по времени *и* частоте одновременно. Спектральный анализ показывает "в ряде есть цикл 20 дней", но не говорит *когда*. Вейвлеты показывают "цикл 20 дней был сильным в марте-апреле, но исчез в мае". Это важно для нестационарных рядов (а финансовые — нестационарны).

**Инструменты:** `scipy.signal` (periodogram, welch), `pywt` (wavelets).

## 7. Стохастические процессы

### 7.1 Geometric Brownian Motion (GBM)

```
dS/S = μ·dt + σ·dW
```

Или в интегральной форме (по лемме Ito): S_t = S_0 · exp((μ - σ²/2)t + σ·W_t).

Это модель Блэка-Шоулза — основа ценообразования опционов и по сей день стандарт индустрии (несмотря на известные недостатки). GBM предполагает: постоянную волатильность, непрерывные пути (без прыжков), нормальные log-returns. Все три предположения неверны для реальных рынков.

**Лемма Ito** — "chain rule" для стохастических процессов. Отличие от обычного дифференцирования — дополнительный член ½σ²·f''·dt, возникающий из-за ненулевой квадратичной вариации броуновского движения. Именно этот член порождает volatility drag: средняя *геометрическая* доходность = μ - σ²/2.

### 7.2 Mean-reversion: процесс Орнштейна-Уленбека

```
dX = θ(μ - X)dt + σdW
```

X стремится к μ со скоростью θ. Ключевые параметры:
- θ: скорость возврата. Большее θ — быстрее возвращается
- μ: долгосрочное среднее
- σ: шум вокруг среднего
- **Half-life** = ln(2)/θ — время полураспада отклонения

Этот процесс — математическая основа pairs trading и statistical arbitrage. Если спред между двумя активами описывается OU, можно торговать: покупать при отклонении ниже среднего, продавать — выше.

**Связь с AR(1):** дискретный аналог OU — авторегрессия первого порядка. X_t = a + b·X_{t-1} + ε. Если |b| < 1 — mean-reverts, если |b| ≥ 1 — нет. Скорость возврата θ = -ln(b)/dt.

### 7.3 Процессы с прыжками

GBM — непрерывные пути. Реальность — прыжки (earnings, news, flash crash). Модель Мертона добавляет пуассоновские прыжки к GBM:

```
dS/S = μ·dt + σ·dW + J·dN
```

Где N — Пуассоновский процесс (прыжки случаются с интенсивностью λ раз в год), J — случайный размер прыжка. Модель лучше описывает хвосты и "implied volatility smile".

### 7.4 Модель Хестона

```
dS = μ·S·dt + √v·S·dW₁
dv = κ(θ - v)dt + ξ√v·dW₂
corr(dW₁, dW₂) = ρ
```

Волатильность — *сама* случайный процесс. Она mean-reverts к θ со скоростью κ, имеет свою "волатильность волатильности" ξ, и *коррелирована с ценой* через ρ (обычно ρ < 0, leverage effect). Хестон воспроизводит volatility smile и кластеризацию волатильности. Используется для ценообразования опционов.

### 7.5 Параметр Хёрста

Обобщение понятия "random walk". Для дробного броуновского движения приращения могут быть *коррелированы*:

- **H = 0.5**: обычное блуждание (независимые приращения)
- **H > 0.5**: persistence (тренд). Если росло — вероятнее продолжит
- **H < 0.5**: anti-persistence (mean-reversion). Если росло — вероятнее развернётся

Связь со стратегиями: если H значимо отличается от 0.5, есть сигнал. H > 0.5 → trend-following. H < 0.5 → mean-reversion. H ≈ 0.5 → рынок случаен на этом горизонте.

**Проблема:** H нестабилен. На одном горизонте рынок может быть persistent, на другом — antipersistent. Более полезен *скользящий* Hurst: H(t), вычисленный на скользящем окне.

**Методы оценки:**
- **R/S анализ** — классический, но ненадёжный (чувствителен к short-range dependence и тренду)
- **DFA (Detrended Fluctuation Analysis)** — устойчивее, убирает тренд
- **Variance ratio** — простой: если var(r_k) / (k·var(r_1)) > 1, H > 0.5

**Инструмент:** можно реализовать самому или использовать `nolds.hurst_rs`, `hurst` пакет.

## 8. Энтропия и информационная теория

### 8.1 Энтропия Шеннона

Энтропия — мера *неопределённости* (или *информации*). Формула для дискретной случайной величины:

```
H(X) = -Σ p(x) · log₂(p(x))
```

Единица — бит. Монетка: H = 1 бит. Кубик: H = log₂(6) ≈ 2.58 бит. Детерминированный исход: H = 0.

**Интуиция для трейдинга:** высокая энтропия = высокая неопределённость = трудно предсказать. Низкая энтропия = предсказуемость = потенциальный edge.

Если рынок — идеальный случайный процесс, энтропия его доходностей максимальна (нет предсказуемых паттернов). Любое отклонение вниз — потенциальная возможность. Скользящая энтропия показывает, когда рынок становится "более упорядоченным" (сильный тренд, кластеризация) — и потенциально более предсказуемым.

### 8.2 Энтропия и финансовые рынки: глубже

**Conditional entropy** H(Y|X) — неопределённость Y, если X известен. H(Y|X) ≤ H(Y), и равенство — при независимости. Разница H(Y) - H(Y|X) = I(X; Y) — *взаимная информация* (mutual information, MI). MI показывает, сколько информации X несёт о Y.

**Для трейдинга:** MI(фича, будущая_доходность) показывает, сколько *любой* (не только линейной) предсказательной информации содержит фича. В отличие от корреляции, MI ловит нелинейные зависимости. Если корреляция ≈ 0 но MI > 0 — зависимость нелинейна.

**Почему MI лучше корреляции для feature selection:**

Допустим, RSI (0-100) влияет на доходность нелинейно: RSI < 20 → рост, RSI > 80 → падение, 20-80 → ничего. Корреляция RSI-доходность будет ≈ 0 (симметричный нелинейный эффект). MI будет > 0, обнаружив информацию. Инструмент: `sklearn.feature_selection.mutual_info_regression`.

### 8.3 Энтропия как мера рыночного состояния

**Низкая энтропия рыночных доходностей** возникает когда:
- Сильный тренд (большинство дней в одну сторону)
- Кластеризация волатильности (предсказуемые режимы)
- Паника (все продают — низкое разнообразие поведения)

**Высокая энтропия:**
- Рэндомный рынок (нет тренда, нет кластеров)
- Высокая ликвидность, много разнонаправленных мнений

На практике: вычисляешь скользящую энтропию (дискретизируешь доходности в k бинов, считаешь Shannon entropy на окне 60-120 дней) и следишь за её динамикой. Резкое падение энтропии → рынок стал предсказуемее → возможность для trend-following. Рост энтропии → рынок "рэндомизируется" → mean-reversion или выход.

### 8.4 Transfer Entropy: кто ведёт, кто следует

Transfer entropy TE(X→Y) — количество информации, которое прошлое X несёт о будущем Y, *сверх того, что несёт прошлое самого Y*.

```
TE(X→Y) = H(Y_t | Y_{t-1:t-k}) - H(Y_t | Y_{t-1:t-k}, X_{t-1:t-k})
```

Если TE(X→Y) > TE(Y→X), то X *информационно ведёт* Y. Это нелинейный аналог Granger causality.

**Применение:**
- Какие активы ведут, какие следуют? (Lead-lag relationships)
- Передача информации между рынками (US → Europe → Asia)
- Обнаружение скрытых зависимостей, невидимых для корреляции

**Проблема:** оценка TE из конечных данных ненадёжна (curse of dimensionality для условных распределений). Нужно *много* данных или осторожная дискретизация/регуляризация.

**Инструменты:** `pyinform`, `JIDT` (Java, с Python wrapper), `IDTxl`.

### 8.5 KL-дивергенция и обнаружение смены режима

KL-дивергенция D_KL(P||Q) — мера "расстояния" между распределениями P и Q:

```
D_KL(P||Q) = Σ P(x) · log(P(x)/Q(x))
```

D_KL = 0 только если P = Q. *Несимметрична*: D_KL(P||Q) ≠ D_KL(Q||P). Симметричная версия — Jensen-Shannon divergence: JSD = (D_KL(P||M) + D_KL(Q||M))/2, где M = (P+Q)/2.

**Для обнаружения смены режима:** вычисляешь KL-дивергенцию (или JSD) между распределением доходностей за последние 30 дней и за последний год. Если JSD резко выросла — распределение изменилось → regime change.

**Для модельной валидации:** D_KL(эмпирическое || модельное) показывает, насколько модель далека от реальности. Если GARCH-модель предсказывает распределение, а реальные доходности сильно отличаются — модель неадекватна.

### 8.6 Сложность Колмогорова и compressibility

Сложность Колмогорова K(x) — длина кратчайшей программы, генерирующей строку x. Невычислима (теорема), но можно приблизить через *сжимаемость*: отношение длины сжатого файла к исходному.

Идея: если ряд хорошо сжимается (gzip, LZ77) — в нём есть структура (паттерны, повторения). Если не сжимается — он рэндомный.

Для финансовых рядов типичная сжимаемость — 30-50%. Это значит, что рынок не полностью случаен (есть структура), но и не детерминирован (далеко от 90%+ сжимаемости). Скользящая сжимаемость — ещё один детектор изменения режима.

### 8.7 Entropy rate и предсказуемость

Entropy rate h — энтропия на символ для стохастического процесса:

```
h = lim_{n→∞} H(X_n | X_1, ..., X_{n-1})
```

Это *фундаментальный предел* предсказуемости: никакой алгоритм не может предсказать лучше, чем позволяет entropy rate. Если h = H(X) (максимум) — процесс iid, прошлое бесполезно. Если h < H(X) — прошлое несёт информацию.

Для финансовых рынков h ≈ H (почти максимум), но чуть ниже — отражает слабую предсказуемость (volatility clustering и другие stylized facts).

## 9. Теория игр и микроструктура рынка

### 9.1 Рынок как игра

Каждая сделка — игра с ненулевой суммой (маркет-мейкер зарабатывает на спреде, биржа — на комиссиях). Но между направленными трейдерами — примерно нулевая сумма: чтобы один заработал, другой должен потерять.

**Равновесие Нэша:** состояние, где ни одному участнику невыгодно менять стратегию. В контексте рынка: если все знают о паттерне, его невозможно эксплуатировать — он уже включён в цены. Вот почему "стратегии из книг не работают" — если паттерн опубликован, он моментально арбитражируется. Alpha живёт в *неравновесных* состояниях — пока рынок ещё не адаптировался.

**Prisoner's dilemma и обвалы.** В кризис каждый участник хочет продать, пока остальные не продали. Индивидуально рациональное поведение (продать) приводит к коллективно иррациональному результату (обвал). Это механизм паники, flash crash, и каждого market sell-off. Для risk management: если ты в crowded trade (позиция, которую держат многие), выход в панику будет *намного* дороже, чем ожидаешь.

### 9.2 Микроструктура рынка

**Order book** — непрерывный двойной аукцион. Покупатели выставляют bids, продавцы — asks. Разница — bid-ask spread, она же минимальная стоимость каждой сделки.

**Три типа участников:**
1. **Маркет-мейкеры** — обеспечивают ликвидность, зарабатывают на спреде. Их враг — adverse selection (торговля с информированными)
2. **Информированные трейдеры** — знают что-то, чего рынок не знает. Торгуют "против" маркет-мейкера
3. **Шумовые трейдеры** — торгуют по причинам, не связанным с информацией (ребалансировка, ликвидность, хеджирование). Обеспечивают маркет-мейкеру возможность зарабатывать

**Market impact** — твой ордер сам двигает цену. Эмпирическая модель квадратного корня: impact ≈ σ · √(V_trade / V_daily). Квадратный корень, а не линейная зависимость — потому что часть impact'а компенсируется ликвидностью. Impact бывает temporary (откатывается после исполнения) и permanent (навсегда сдвигает цену). Permanent impact — информационный: твоя торговля *раскрывает* информацию рынку.

**Alpha decay:** чем больше участников знает о стратегии, тем меньше alpha. Capacity стратегии — объём капитала, при котором market impact съедает всю alpha. Для большинства стратегий capacity = $1-100M. Для простых — ещё меньше. Это объясняет, почему Renaissance Technologies ограничил Medallion Fund ~$10B и вернул деньги внешним инвесторам — capacity ограничена.

### 9.3 Optimal Execution

Когда нужно купить/продать большой объём, как минимизировать impact?

**TWAP** (Time-Weighted Average Price) — делим равномерно по времени. Просто, но не учитывает рыночные условия.

**VWAP** (Volume-Weighted Average Price) — торгуем пропорционально историческому профилю объёма (больше утром и вечером, меньше в обед). Лучше TWAP, но предсказуем.

**Almgren-Chriss** — оптимальный баланс между market impact (торгуй быстро) и market risk (торгуй медленно, рискуя неблагоприятным движением цены). При высоком risk aversion — торгуй быстро (ближе к market order). При низком — растяни (ближе к TWAP).

## 10. Риск-менеджмент

### 10.1 Почему это важнее стратегии

Цитата Лари Хайта (одного из пионеров алготрейдинга): "Если ты рискуешь на каждой сделке долей, которая может тебя убить, ты *умрёшь*. Неважно, насколько хороша стратегия."

LTCM имел Sharpe ≈ 4.3 (до leverage). Потерял 90% за 4 месяца. Knight Capital имел прибыльный алгоритм. Потерял $440M за 45 минут из-за бага. Amaranth имел правильный view на природный газ. Потерял $6B из-за чрезмерной позиции. Во всех случаях проблема — не стратегия, а risk management.

### 10.2 Value at Risk и его недостатки

**VaR(α)** — "С вероятностью (1-α) потери за день не превысят X."

VaR(95%) = -$1M означает: в 95% дней потери < $1M. Но VaR *ничего* не говорит о том, что происходит в оставшихся 5%. Потери могут быть $1.1M или $100M — VaR не различает.

**VaR не субаддитивен.** VaR(A + B) может быть > VaR(A) + VaR(B). Это значит: по VaR диверсификация может *увеличить* риск. Это не просто теоретический курьёз — это нарушение базовой интуиции о том, как работает риск.

Три способа считать VaR:
- **Исторический** — берём α-перцентиль исторических доходностей. Просто, но привязан к прошлым данным
- **Параметрический** — предполагаем распределение (обычно нормальное), считаем аналитически. Быстро, но нормальное занижает хвосты
- **Monte Carlo** — генерируем сценарии из модели, считаем перцентиль. Гибко, но зависит от модели

**Cornish-Fisher VaR** — поправка к параметрическому VaR, учитывающая skewness и kurtosis. Существенно лучше нормального при fat tails.

### 10.3 CVaR (Expected Shortfall)

**CVaR(α)** = E[loss | loss > VaR(α)]. "Средний убыток в те дни, когда убыток превысил VaR."

CVaR — *когерентная* мера риска (субаддитивна). CVaR всегда ≥ VaR. Для нормального распределения CVaR(95%) ≈ 1.28 × VaR(95%). Для fat tails — значительно больше (что отражает реальность).

Регулятор (Basel III) перешёл от VaR к CVaR именно потому, что CVaR лучше отражает хвостовые риски.

### 10.4 Position Sizing: критерий Келли

Келли отвечает на вопрос: "Какую долю капитала ставить на каждую сделку, чтобы максимизировать долгосрочный рост?"

**Формула для бинарных исходов:** f* = (p·b - q) / b, где p — вероятность выигрыша, q = 1-p, b = средний выигрыш / средний проигрыш.

**Для портфеля:** f* = Σ⁻¹(μ - r_f), где Σ — ковариационная матрица, μ — вектор ожидаемых доходностей. Это Markowitz с конкретной целевой функцией (max log-wealth вместо max Sharpe).

**Свойства Келли:**
- Максимизирует *геометрическую* доходность (рост в длинной серии)
- Минимизирует время достижения целевого капитала
- Но... волатильность при full Kelly *огромна*. Просадки 50-80% — норма

**Почему Half-Kelly (или Quarter-Kelly):**
Функция роста как функция размера ставки имеет форму: growth(f) = p·log(1 + f·b) + q·log(1 - f). Эта функция пологая около максимума — при f/2 теряешь только 25% роста, но снижаешь волатильность вдвое. При f*1.5 рост *падает* (over-betting хуже under-betting!). При f*2 — рост = 0. При f > 2·f* — ожидаемое разорение.

Практическое правило: используй Half-Kelly или меньше. Ошибка в оценке параметров (p, b) при Full Kelly может превратить прибыльную стратегию в убыточную. При Half Kelly система устойчива к ошибкам оценки.

### 10.5 Drawdown

Max drawdown (MDD) — максимальное падение от пика до дна. Для трейдера это, пожалуй, *самая* важная метрика: "Насколько плохо может быть?"

**Ожидаемый max drawdown** (для GBM): E[MDD] ≈ √(π/2) × σ × √T. Для Sharpe = 1 на 10-летнем горизонте: E[MDD] ≈ 15-25%. Но *фактический* MDD имеет огромную дисперсию — может быть в 2-3 раза хуже ожидаемого.

**Время восстановления:** часто важнее, чем глубина. Drawdown -20%, из которого выходишь за 3 месяца — терпимо. Drawdown -20%, из которого выходишь за 3 года — мучительно.

**Underwater curve** — визуализация текущего drawdown во времени. Показывает, сколько времени стратегия проводит "под водой" (ниже предыдущего пика). Для типичной стратегии с Sharpe 1 — около 50% времени.

## 11. Оптимизация портфеля

### 11.1 Markowitz: красивая теория, проблемная практика

Mean-Variance Optimization (MVO): максимизируй E[r] - γ/2 · σ². Решение: w* = (1/γ) · Σ⁻¹ · μ.

**Проблема "Garbage In, Garbage Out":** MVO экстремально чувствительна к входным данным. Небольшое изменение в оценке ожидаемых доходностей → радикально другие веса. А ожидаемые доходности оцениваются с огромной ошибкой (см. закон больших чисел). Michaud назвал MVO "error maximizer" — она концентрирует портфель в активах с *наибольшей ошибкой оценки*.

### 11.2 Альтернативы

**Minimum Variance** — оптимизируй только по риску (без μ). Веса зависят только от ковариационной матрицы, которая оценивается надёжнее, чем ожидаемые доходности. На практике часто работает лучше "полного" MVO.

**Risk Parity** — каждый актив вносит равный *вклад в риск* портфеля (а не равный вес). Портфель 60/40 (акции/облигации) выглядит диверсифицированным, но ~90% риска — от акций. Risk Parity уравнивает вклады: меньше акций, больше облигаций (с leverage для достижения целевой доходности). Bridgewater All Weather — самая известная Risk Parity стратегия.

**Black-Litterman** — байесовское решение проблемы входных данных. Начинаем с "рыночного равновесия" (implied returns из рыночных капитализаций), затем добавляем свои "views" с указанием уверенности. Если view слабый (низкая уверенность) — веса почти не меняются. Если сильный — двигаются. Элегантное решение проблемы "garbage in".

**Hierarchical Risk Parity (HRP)** — метод Lopez de Prado. Использует hierarchical clustering для разбиения активов на кластеры, затем аллоцирует между кластерами и внутри них. Не требует обращения ковариационной матрицы (работает при N > T, когда MVO невозможна). На практике часто лучше MVO.

**Инструменты:** `cvxpy` (convex optimization), `riskfolio-lib` (все методы портфельной оптимизации), `PyPortfolioOpt`.

## 12. Машинное обучение: ожидания vs реальность

### 12.1 Почему ML в финансах — это другой мир

В CV/NLP: много данных, стабильные паттерны, отсутствие adversary. В финансах: мало данных (252 дня/год), нестационарные паттерны (что работало вчера — не работает завтра), adversary (другие ML-модели торгуют *против* тебя и съедают alpha).

**Signal-to-noise ratio** в финансах ничтожен. Лучшие стратегии предсказывают направление с точностью 51-53%. ML-модель, обученная на 5 годах дневных данных, имеет ≈ 1260 наблюдений. Для CV это смешно — ImageNet имеет миллионы. Для финансов 1260 — *хороший* датасет.

При таком объёме данных и слабом сигнале overfitting — *дефолтный* исход. Модель, которая "работает", скорее всего заучила шум. Бремя доказательства — на тебе: нужно убедительно показать, что это не overfitting.

### 12.2 Правильная валидация

**Time-Series Split:** train на [0, T1], test на [T1, T2], повторить с расширением. Никогда не shuffle, никогда не random split. Будущее не должно протекать в прошлое.

**Embargo period:** зазор между train и test (например, 5 дней), чтобы предотвратить leakage через overlapping features (скользящие средние, лаги).

**Purged Cross-Validation (Lopez de Prado):** для labels, создаваемых из overlapping windows (например, "доходность за следующие 5 дней" — labels для соседних дней перекрываются). Purged CV убирает из train все наблюдения, чьи labels пересекаются с test period.

**Walk-Forward Analysis:** поезд → тест → сдвинь окно → повтори. Самый реалистичный метод. Показывает, как модель работала бы в реальном времени.

### 12.3 Feature engineering важнее модели

В финансах 80% успеха — правильные фичи, 20% — модель. Gradient boosting на хороших фичах побьёт deep learning на плохих.

**Fractional differentiation** — превращает нестационарные ценовые данные в стационарные, сохраняя информацию (см. главу 6).

**Triple Barrier Labeling** — вместо "цена через 1 день" используем "какой барьер сработал первым: take-profit, stop-loss, или max holding period?" Это создаёт более осмысленные метки, учитывающие реальную торговую механику.

**Meta-labeling** — двухуровневая система: первая модель генерирует направление (buy/sell), вторая решает — *следовать ли* сигналу и с каким размером. Вторая задача проще — предсказать качество сигнала легче, чем направление рынка.

### 12.4 Когда ML помогает, когда нет

| Задача | ML полезен? | Комментарий |
|--------|-------------|-------------|
| Прогноз направления по ценам | Слабо | SNR слишком низкий |
| Прогноз волатильности | Да | Есть сильный сигнал (clustering) |
| Feature selection | Да | MI, SHAP, permutation importance |
| Regime detection | Да | HMM, clustering |
| NLP/sentiment | Да | LLM лучше всего |
| Альтернативные данные | Да | Satellite, web traffic, text |
| Anomaly detection | Да | Изоляция нестандартных паттернов |
| Optimal execution | Средне | Reinforcement learning, но данных мало |

### 12.5 LLM в торговле

Самое полезное применение LLM — *не* предсказание цен. LLM:
- **Sentiment analysis:** лучше любых rule-based систем. Понимает контекст, сарказм, финансовый жаргон. Claude/GPT API с правильным промптом → numerical sentiment score
- **Earnings call analysis:** извлечение structured data из текста (guidance, ключевые метрики, тон менеджмента)
- **Генерация гипотез:** "Какие факторы могут влиять на spread между X и Y?"
- **Code review:** "Проверь мой бэктест на look-ahead bias" — LLM отлично находит subtle bugs
- **Объяснение:** "Объясни, почему EGARCH лучше GARCH для этих данных"

LLM *не* предсказывает рынок. Это языковая модель, а не oracle. "Куда пойдёт Bitcoin?" — бессмысленный вопрос к LLM.

## 14. Чеклисты и war stories

### 14.1 От идеи до live trading

```
═══ ГИПОТЕЗА ═══
□ Конкретная тестируемая формулировка
□ Экономическое обоснование (кто теряет деньги?)
□ Edge > transaction costs?

═══ ДАННЫЕ ═══
□ >3 лет (daily), >6 мес (intraday)
□ Включает bull, bear, sideways, crisis
□ Point-in-time (без будущих пересмотров)
□ Train/val/test split ДО анализа

═══ BACKTEST ═══
□ Нет look-ahead bias
□ Реалистичные costs (комиссии + спред + slippage + impact)
□ Walk-forward validation
□ Стабильность при ±20% параметров
□ Deflated Sharpe (с учётом N тестов)

═══ СТАТИСТИКА ═══
□ Sharpe > 1.0 (после costs)
□ Bootstrap CI для Sharpe не включает 0
□ Max drawdown приемлем
□ Profit factor > 1.5

═══ RISK ═══
□ Position sizing (≤ Half-Kelly)
□ Stop-loss + kill switch
□ Стресс-тест (1987, 2008, 2020)
□ Correlation с существующими стратегиями

═══ LIVE ═══
□ Paper trade 1-3 месяца
□ Начинаем с 10-25% капитала
□ Мониторинг + alerts на аномалии
□ Ежеквартальная ревалидация
```

### 14.2 War stories

**LTCM (1998).** Sharpe ≈ 4.3. Leverage 25:1. Модели предполагали нормальные распределения и стабильные корреляции. Россия дефолтнула → flight to quality → все спреды разошлись одновременно → корреляции → 1 → margin calls → принудительная ликвидация → спреды ещё шире (positive feedback). Потеря: $4.6B (из $5B). Урок: (1) fat tails реальны, (2) корреляции в кризис → 1, (3) leverage убивает, (4) ликвидность исчезает когда нужна больше всего.

**Knight Capital (2012).** Деплой нового кода, тестовый флаг не удалён на одном сервере. Алгоритм торговал *против* рынка 45 минут. Потеря: $440M. Урок: kill switch обязателен, мониторинг позиций в реальном времени, max loss per day, canary deployment.

**Gaussian Copula и кризис 2008.** Дэвид Ли предложил формулу для оценки корреляции дефолтов в CDO. Формула использовала Гауссову копулу, у которой tail dependence = 0. Это означало: "одновременный дефолт многих заёмщиков практически невозможен." Вся индустрия использовала эту формулу. Когда рынок недвижимости упал, дефолты оказались *сильно* коррелированы в хвостах — именно то, что Гауссова копула не может уловить. Потери: триллионы. Урок: tail dependence matters, и "все используют модель" ≠ "модель правильная."

**Renaissance Technologies.** Medallion Fund: ~66% годовых (до комиссий) 30+ лет подряд. Sharpe ≈ 6 (gross). Как: (1) огромное количество слабых сигналов, не один сильный, (2) короткий holding period, (3) уникальные данные (weather, satellite, unusual sources), (4) строжайшая статистическая дисциплина, (5) PhD в физике/математике/CS (не в финансах), (6) инфраструктура мирового уровня, (7) capacity ограничен ~$10B. Урок: это возможно, но требует исключительных ресурсов и дисциплины.

### 14.3 Иерархия важности

```
1. Риск-менеджмент (без него всё остальное бессмысленно)
2. Transaction costs (стратегия прибыльна ПОСЛЕ комиссий?)
3. Статистическая валидация (это не overfitting?)
4. Простая модель (линейная регрессия, z-score, GARCH)
5. Данные (quality > quantity)
6. Сложные модели (ML) — ТОЛЬКО если простые не хватает
7. Скорость — для большинства стратегий не bottleneck
```

### 14.4 Формулы — шпаргалка

```
ДОХОДНОСТИ
  Simple:          r = (P₁ - P₀) / P₀
  Log:             r = ln(P₁ / P₀)
  Annualize:       r_ann = r_daily × 252,  σ_ann = σ_daily × √252
  Vol drag:        geo_return ≈ arith_return - σ²/2

PERFORMANCE
  Sharpe:          √252 × mean(r) / std(r)
  Sortino:         √252 × mean(r) / downside_std(r)
  Calmar:          annual_return / |max_drawdown|
  Profit Factor:   Σ(wins) / |Σ(losses)|
  Expectancy:      win_rate × avg_win - loss_rate × avg_loss

RISK
  VaR(α):          Percentile(returns, α)
  CVaR(α):         E[r | r < VaR(α)]
  Kelly:           f = (p·b - q) / b,  portfolio: f = Σ⁻¹(μ - rᶠ)

TIME SERIES
  OU half-life:    t½ = ln(2) / θ
  Hurst:           H > 0.5 trend, H < 0.5 mean-revert, H = 0.5 random
  GARCH:           σ²ₜ = ω + α·ε²ₜ₋₁ + β·σ²ₜ₋₁

STATISTICS
  SE(Sharpe):      √((1 + SR²/2) / n)
  Min track record: n ≈ (z/SR)² × (1 + k/4 · SR²)

MICROSTRUCTURE
  Market impact:   I ≈ σ × √(V_trade / V_daily)
```
